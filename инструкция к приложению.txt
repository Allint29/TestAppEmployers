Инструкция к приложению "Учет сотрудников".

Подключение к MSSQL через конфигурацию:
"ConnectionStringsToMyDbContext": {
    "MyDbContext": "Server=DESKTOP-ALBME3B;Integrated security=SSPI;database=TestingDATABASE;"
  },
Подключался через Integrated security=SSPI, пароль не учитывал при настройке строки соединения, 
поэтому ЧЕРЕЗ ПАРОЛЬ РАБОТАТЬ НЕ БУДЕТ (метод CreateNewDatabase() в Services/MyDbContext.cs), 
конфигурацию с логином паролем не делал, так как проверить не на чем.

Работа приложения:
1. При запуске - открывается страница, 
	Eсли БД нет на сервере:
	а)Происходит создание БД (3 Таблицы со связями один ко многим, одно представление 
	со всеми данными из всех таблиц, 15 хранимых процедур)
	б)Создаются автоматом первые записи в таблицах Должностей и Отделов
	Если БД есть:
	а)Просто происходит подключение (ругательства на то что процедуры и таблицы есть 
	ловяться catch и просто пишуться в дебаг)
	
2. На главной странице отображаются сотрудники.

3. Есть кнопки для создания автоматически-сгенерированных сотрудников(чтобы не вводить руками).

4. Есть кнопки создания Должностей, Сотрудников, Отделов (ручной ввод). У каждого открывается свое окно.

5. Напротив каждого сотрудника ссылки на редактирование и удаление записи - для перехода на страницы
	подтверждения действий.
	
6. Реализован поиск по ФИО, должности, отделу регистр не учитывается. Для поиска нажать кнопку "Искать"

7. Есть фильтры по отделам и должностям. Для фильтра нажать кнопку "Искать".

8. Реализован механизм валидации данных при создании, редактировании.

9. Увольнение сотрудника реализовано на отдельной форме (необходимо ввести дату и отметить нужный чекбокс
	инача валидацию не пройдет). При увольнении кнопка "Уволить" в карточке редактора сотрудника скрыта.
	
10. В основном все действия беруться из сгенерированных хранимых процедур. (Есть несколько больших 
	запросов,  которые просто прописаны в коде - для экономии времени).
